Common File System Elements

Permissions button: This is the button you click to add/remove/modify permissions for users.
Folder: A container for files that can also contain other sub-folders.
File: Unit of storage that holds data or information and is identifiable by a unique name and path
User: Individual or entity that interacts with the file system
User Group: A collection of Users

Inheritance

Files and folders can inherit permissions from their parent folders. This inheritance can be turned off for specific files/folders.

Key Principles:

  • Direct permissions take precedence over inherited permissions.
  • Permissions closer to the file take precedence over those further away.
  • "Deny" permissions override "allow" permissions at the same level.
  • User and group permissions have the same priority.

    • Resolving Permissions Issues:

  • When a user tries to access a file, Windows looks at all direct permissions for that user or their groups.
  • If any "deny" permissions are found, access is denied.
  • If no "deny" permissions are found, but there are "allow" permissions, access is granted.
  • If no explicit permissions are found, and inheritance is enabled, Windows looks at the parent folder's permissions.
  • If no relevant permissions are found after checking inheritance, access is denied.
